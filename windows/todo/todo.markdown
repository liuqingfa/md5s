# 状态模式

当控制一个对象状态转换的条件表达式过于复杂，将其判断的逻辑转移到表示不同  
状态的一系列类当中去，可以把复杂的逻辑简化。

将特定状态相关的行为局部化，并将不同状态的行为分割，可通过定义新的子类  
来增加新的状态和转换；通过把各种状态转移逻辑分布到子类中从而减少相互间  
的依赖。

# 状态模式

当一个对象的行为取决于它的状态，并且它必须在运行时根据状态改变行为的时候，  
就可以考虑使用状态模式了。

状态模式基于对象内部状态变化来控制使用行为  
策略模式定义多个行为并进行封装，针对策略使对象使用不同行为。

# 装饰模式

装饰模式可以让类在运行时改变自身的行为，增加或减少，区别于继承的编译时增加  
代理模式为解决类之间调用的耦合，作为类的中间件方便复用和隐蔽，编译时生效
